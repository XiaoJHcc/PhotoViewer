## 1. M1 步骤与 GATE

### 1.1 数据可读性前置确认（快速闸，部分已确认）

历史星级是这套训练数据的标签来源，开工前确认能读到。

- **已确认**：指纹聚合可靠（同次曝光 RAW/HEIF/JPG 合一）、各题材库与"金标准"的相对配比大致对应（plan-3-0 §0.4 / §1 决策 11）。
- **仍需抽样确认**：人工星级确实写在 **XMP / 同名 `.xmp` sidecar** 上、能被读到（ARW 的 rating 通常在 sidecar，不在文件内）；拍摄时间字段齐全（`DateTimeOriginal` / `SubSecTimeOriginal`，指纹与拍摄段切分都依赖）。用 `Tools/ExifTestTool` 抽查，**重点覆盖老批次**（跨年历史文件可能改名 / 缺 SubSec / sidecar 与文件时间不一致）。
- **GATE**：星级读不到（只在 Lightroom / Capture One 的 catalog 里）→ **先批量导出 XMP / sidecar**，否则后续全部无从谈起。

### 1.2 照片增强制式探针 + 特征可行性（**当前要执行的第一步**）

> 这一步合并两件事：① 选定喂 DINO 的**输入制式**（仅原片 / 仅增强 / 多视图）——它决定 1.3 入什么、特征意味着什么，必须在全量入库前定；② 同时验证 backbone 特征对"段内细差"够不够敏锐（原线性探针）。依据见 plan-3-0 §0.3（雾 / 低对比片标签建立在 ACR 增强视图上）、§0.7、§1 决策 13。

**已确定（本次讨论收敛，记录在此、不再反复）：**

1. **目标是"兼得"**：构图可辨（尤其雾 / 低对比片，记为 **A**）+ 影调理解（天生好影调要认得出，记为 **B**）。实现机制 = **多视图输入**（原片 + 确定性增强，两路 CLS 拼接）：原片路保 B、增强路补 A。
2. **不用**不变性增广（会教模型忽略影调 → 杀 B）、**不用**调色版当更高星新样本（OOD）。
3. **铁约束**：增强必须是**确定性算法、推理期对每张可复现**；**手动 ACR 修图只作调参参照、不作输入**（production 里不可能每张手修——精修像素同理，见 plan-3-0 §0.4）。
4. **判据 = DINO 判别力（线性探针准确率），不是主观好看**；普通可比对集里要**专门含"靠影调 / 氛围区分"的对**，否则测不出"仅增强"有没有伤 B。
5. **增强产品化、算法探索搬进产品**（落地 plan-3-0 §1 决策 13 末句"产品化待本节"）：增强既是模型输入视图、又是查看器产品功能，二者共用同一确定性算法；算法的探索与目视评估在产品内做，而非纯 Python 脚本里盲调。

**产品化落地（算法探索的载体）：**

- **入口**：图片查看器**控制栏新增一个 toggle 按钮**，点击即把主画面原片**原地替换**成增强版本，再点还原。落点在 `ViewModels/Main` 的控制 VM（`ControlViewModel`）+ 图片 VM（`ImageViewModel`），增强是一步**确定性图像处理**（输入原片解码位图 → 输出增强位图），不入库、不动原文件。
- **已落地算法（本次收敛，覆盖原"v0 粗暴均衡→逐步换 CLAHE"设想）**：定位是**信息归一化器**——不追求好看成片，而是把有效信息分布得更均匀、对比更清晰但**低失真**，供模型 / 人眼判断"后期潜力"（真正后期由人做）。技术选型 = **全局限对比直方图均衡（CLHE）**：亮度直方图按 `ClipFactor` 裁剪封顶 + 均匀回填（封住 LUT 斜率 → 防断层、防大面积同色尖峰独吞输出范围）→ cdfMin 归一化 LUT → 按 `(newY+ε)/(oldY+ε)` 等比缩放保色相（ε = 暗部增益护栏，防夜景彩噪）。落点 `Core/Image/ImageEnhancer.cs`，纯算法、不入库、不动原文件。
- **为什么是 CLHE 而非局部 / 分段 / 自适应**：选型第一约束是**可学习性 5 规格——全局、单调、平滑(斜率有界)、少量固定参数、色彩稳定**，使"原片↔增强片"是一致的全局单调色调变换、模型能建立关联。据此**显式排除**真分块 CLAHE（局部自适应 → 同输入随位置出不同值）、双直方图分段（拐点 / 切点逐图跳）、逐图自适应强度（再叠一层逐图变化）——它们都更"漂"、伤可学习性；裁剪一步即覆盖"匀信息但不过度摊平"的目标。
- **可调参数（固定常量，不进设置）**：`ClipFactor = 2.0` 是唯一手感旋钮 = **保守↔激进**：→1 近线性最稳、↑ 对比更强、≳4 退回裸 HE（断层 / 漂移回潮），实用区间 **1.5–3**；`ChromaGainEpsilon = 6` 暗部增益软上限。
- **目视评估（本次结论）**：雾天灰片显著增强（人眼 / DINO / CV 提取均更足）；四类直方图（中部单峰 / 极灰窄峰 / 偏态 / 双峰）均表现良好，无断层、无亮度漂移、无暗部彩噪。
- **铁约束（承 plan-3-0 §1 决策 13，不变）**：CLHE 是**产品 / 目视侧已收敛**的信息归一化算法，也是模型输入的**当前首选候选**；但是否采纳为模型输入仍受下面**线性探针 ≥80% GATE + 制式决策**约束；产品 ship 版与模型输入版须收敛到**同一非破坏性算法**。

**本次目视验证 + 入模前提（统一认知，本次讨论收敛）：**

- **A（构图可辨）目视已坐实**：DINO 诊断图上，原本埋在雾里的远处桥 / 建筑，增强后与海面、山体分离开 → 增强对 A 有实效。但这是**先行指标**（patch 级、近 N=1 目视），不等于探针达标（CLS 级、定量 ≥80%）。
- **B（影调理解）目视沉默、仍待探针**：桥分离只证 A，对"会不会把天生好影调与被救回来的灰片抹平"这条 B 风险一字未答；探针样本须**专含影调对**才测得出。
- **产品价值 ≠ 模型价值，二者可分离**：增强对**人眼选片**的价值基本坐实（作为控制栏 toggle 留用，与是否入模无关）；增强对**模型**有无增量是独立问题，可能结论是"原片已够"。
- **制式（架构倾向 vs 数据出口）**：架构上倾向**原片 + 增强多视图**（保 B 补 A）；但**保留本节既定出口"仅原片在低对比上已够分 → 不增强、收工"**——增强这一路是否真要入模，由探针 ≥80% + 双视图成本 / 收益（双 DINO pass、双存储、端侧成本）判，不预设。
- **入模前一致性清单（执行项，非决策）**：① C# 产品侧与 Python 探针 / 入库**逐像素一致**；② 增强施加的**解码阶段 / 分辨率训练-推理处处同一步**（产品跑全分辨率、DINO 走 560 缩略图，直方图随分辨率变 → LUT 变）；③ 多视图的**双 CLS schema / 存储 / 端侧代价**；④ 参数（`ClipFactor=2.0` / `ε=6`）**冻结进模型契约**，改则重提整库；⑤ 过线性探针 **≥80%（含影调对）**。

**量化判据（产出 `Tools/feature_probe.py`）：**

- 取 ~200 张含多组近重复 + 一组雾 / 低对比片的样本，入库拿 CLS（ViT-S/16@518，384d）。
- **第 0 步目视已搬进产品**：用控制栏 toggle 在真实雾片上直接切「原图 ↔ 增强」对比，确认"均衡 ≠ ACR 效果"（均衡多半放大噪声、显得生硬）→ 据此定出送进下面矩阵的候选算法（不必再在脚本里并排出图）。
- **矩阵**：输入制式{仅原片 / 仅增强 / 多视图} × 算法{产品内目视筛出的候选，如 CLHE / 去雾·CLAHE} × 集合{普通段内可比对（含影调对）/ 低对比雾片可比对}；在"人眼可分但星级不同"的段内可比对上，用逻辑回归 / 线性 SVM 对 CLS **差异向量**做二分类（判谁星级更高）。增强算法须与产品 toggle **同一份实现**（C# 产品侧与 Python 探针侧逐像素一致，否则探针结论不迁移到产品 / 入库）。
- **基座判据**：最佳制式准确率 **≥ 80%** → ViT-S 够用；**< 80%** → 升 ViT-L（或 518→1024）复测，仍不行再排查特征本身（决定 plan-3-0 §1 决策 8 落点）。
- **制式决策规则**：
  - 仅原片在低对比上已够分 → **不增强**，收工。
  - 否则 → **多视图**（保 B 补 A，兼得的形态）。
  - **仅增强默认不选**——通常牺牲 B（把"天生好影调"和"被救回来的"拉成一样），除非探针显示它在"靠影调区分的对"上不掉点。
  - 全都没用 → 分辨率 / 特征问题，升 ViT-L，而非折腾预处理。
- 产出：`feature_probe.py` + 一页结论（矩阵准确率表 + t-SNE 目视：近重复是否聚簇、跨题材是否分离）。

**待你明确（后续补充后再执行）：**

- **具体增强算法与参数**：**已定** = CLHE + `ClipFactor=2.0`（见上"产品化落地"）；**自适应强度已否决**（逐图变化伤可学习性，改固定常量）。仅余可选项：若要"均匀加对比"的全局观感，可在 CLHE 后叠一条固定 S 曲线 / gamma（同样规律、待需要再加）；最终是否入模仍待探针。
- **产品化思路**：toggle 是否记忆每图状态、是否提供强度档位；多视图在推理期的代价（双 DINO pass、`photos.db` 是否存两路特征、端侧成本），以及是否全库都用多视图、还是仅对判定为低对比的片用。
- **探针样本**：雾 / 低对比片正反例各几张（你提供），并入上面 ~200 张。

- **GATE**：制式选定 + 基座达标 + 产品 toggle 上真实雾片目视确认增强可用 → 1.3；不达标 → 升 backbone 再来。

### 1.3 选子集 + 全量入库

- 按**题材规则**圈定训练 / 测试子集（plan-3-0 §1 决策 11；配比由你定，控泛化 / 特化）。开工前先确认：
  - 磁盘余量（两万张 patch ≈ 30GB）；
  - 库目录的**拍摄日期 / 拍摄段可恢复**（按段 / 日划分、滑窗都依赖它）。
- **入库视图数由 §1.2 制式结论决定**：若选"多视图"，需对原片与增强视图各提一次 CLS（schema / 存储按 §1.2 产品化结论定）；若"仅原片"，维持现状。
- 跑现成 `Tools/BatchIngest`（DirectML 加速）批量入库 → 填满 `photos.db`。
- **GATE**：抽查入库完整性——指纹聚合正确（同次曝光 RAW/JPG/HEIF 合一）、EXIF / rating 覆盖率达标、CLS / patch / cv_grid 三路齐备。

### 1.4 data_audit（看清真实分布、校准所有阈值）

新建 `Tools/data_audit.py`，读 `photos.db` 输出（这是 M2+ 一切阈值的来源）：

| 统计项 | 用途 / 校准什么 |
|---|---|
| 星级分布直方图（总体 + 每段） | 确认锦标赛结构；验证"每段分布大致相同"（plan-3-0 §0.3） |
| CLS cosine 全库直方图 + 分位 | 校准相似度→权重映射、tie 上限 |
| 段内 vs 跨段 cosine 分布 | 验证滑窗 + 相似度能否分离可比对 |
| 拍摄段切分结果（按时间间隔 / 相似度） | 看每段张数、段时长，定窗口大小 |
| 近重复对（cosine ≥ 0.98）的 CV 距离分布 | 校准"条件保留"的 CV 差异阈值（决策 1） |
| EXIF / rating 字段覆盖率 | 确保 ≥ 95%，定缺失策略 |
| ≥3★ 实际占比 | 校准"Top 12.5%"是否对得上真实分布（不对则调召回口径） |

- **GATE（M1 总闸）**：星级可读 + 特征敏感 + 数据覆盖达标 + 分布无异常 → **M1 完成，进 plan-3-2（M2）**；任一异常 → 停下排查，不带病进训练。

## 3. 后续里程碑（简述，随 M1 结果调整，细节不锁）

> 只记方向与 GATE，承 plan-3-0 §1 已锁定技术决策；具体参数、配对量、架构待 M1 audit 后在 plan-3-2 起逐一展开。

- **M2 跨段绝对校准（偏移法）**：拍摄段切分 → 每段抽代表 + 精修锚点入"绝对池"跨组评级 → 拟合每段偏移 → 产出绝对星级（连续、不 clamp）。**GATE**：偏移自洽（重复评级一致）+ 跨天"同绝对级"能对齐；否则退回纯相对、绝对留 v2。
- **M3 训练对生成**：段内对（高权，相似度加权 + 条件保留 + 大星级差降权）+ 跨段对（绝对星级派生、**低权**）；候选组为单位存储；按段 / 日划 split。
- **M4 基线 + 绝对涌现测量**：段内 ranking 立地基（段内 top-1、段内 0-5 准确率）；金标准 / 锚点测**绝对涌现**（好/坏天气片落位是否合理、锚点是否落顶端）→ **a/b/c 决策点**（绝对够 → 进 M6；不够 → M5）。
- **M5 绝对校准增强（条件触发）**：绝对不够才加大偏移监督权重 / 补少量 targeted 跨段标注；重测绝对涌现。
- **M6 系统集成 + 全谱评估**：模型 + 去重压低算法 + 窗口召回；两条线评估（≥3★ 召回 + 0-5 准确率）+ 七类场景抽检 + 金标准集。**GATE**：达标 → M7，否则失败分析回退。
- **M7 部署**：ONNX 导出 + `PhotoViewer/Core/AI/` C# 推理集成（A2 已打通端侧链路，主要是换最终模型）。

## 5. 并行长周期任务：金标准验证集（建议 M1 期间就开始）

M4 起的评估、a/b/c 决策都以它为准绳（plan-3-0 §0.6 称其"唯一可信标尺"），且只能人工精选、周期长——**别等到 M4 才攒**。

- 规模 ~50-100 组，按 plan-3-0 §0.5 七类场景分布。参考构成：
  - ~30 组**近重复**（每组 3-8 张，标出最佳 1-2 张）
  - ~20 组**一般场景**（标 ≥3★ 该保留哪些）
  - ~20 张**孤立片**（标是否值得进人工二审）
  - ~10 组**极相似**（标"不可分 / tie"）
- 另含**精修精品锚点**（plan-3-0 §0.4）：既是金标准顶端，也是 M2 偏移法绝对池的成员。
- 这套集子上算的 0-5 准确率 / 组内 top-1 / 七类场景表现，比跑在带噪星级上的自动指标可信得多。

## 6. 后续里程碑的初期想法（不锁，供 plan-3-2 起草用）

> 对话中成形、但需 M1 数据 / 审计后才能定稿的想法，先记下免得丢失。**均未锁定**，plan-3-2 起按真实分布重定。

### 6.1 M2 评分偏移法（跨段绝对校准）

- **粒度**：偏移按**拍摄段**给（不按日历日）；段切分用 plan-3-0 §1 决策 2 的"时间间隔 + 相似度"那套，与可比域同源。
- **估偏移的办法**：每段抽几张代表，连同**精修精品锚点**一起丢进一个**绝对池**做一次跨组评级（小型 meta-锦标赛）；某段代表在绝对池里的级别 − 它原本的段内级别 = 该段偏移。精修片天然钉住高端。
- **刻度**：偏移后用**连续刻度、不 clamp 回 0-5**（好天 5★+2=7、坏天 0★−2=−2 是对的，表达更宽的真实绝对范围）。
- **怎么喂训练**：段内对继续用**原始段内星级、高权**（可靠地基）；跨段对用**偏移后绝对星级派生、低权**（粗信号）。
- **诚实边界**：偏移是手工注入的绝对判断、±2 很粗，质量上限卡在这——它是**锚不是真值**。M2 GATE 必须验：重复评级是否自洽、跨天"同绝对级"能否对齐；不行就退回纯相对、绝对留 v2。

### 6.2 M4 单图编码器架构初步想法

> 承 plan-3-0 §1 决策 6（纯单图）、决策 7（先线性探针）。具体维度 / 层数待 M1 探针 + audit 后定。

- **三路融合的核心概念**（来自一 / 二期）：**patch token 作为 CV 网格的 per-cell 语义权重**——每格"该不该锐"由 patch 语义定（主体格重、背景格轻），每格"实际多锐 / 多抖"由 CV 物理量给。这正是存 patch token 的理由，对应场景 6（题材化权衡）。**不**在架构里硬编码"主体清晰度权重 > 背景"，让 MLP 自己学。
- **单图编码起点**（轻量优先、防过拟合）：CLS(384) + CV 网格按 patch 主体权重做加权统计（每标量的加权 mean / std / 分位 + 最差格）+ EXIF 向量 → 小 MLP → score。先跑这个轻量版；若失败集中在"局部抖动 / 虚焦"（空间信息丢失）再上保留更多空间结构的变体。
- **EXIF 向量化**：对数化再标准化（log 焦距×crop、log 光圈×crop、log 快门、crop），缺失填 0 + mask 位；约 4-8 维。
- **判断优先级的架构含义**（plan-3-0 §0.6）：构图 → 内容 → 光影；构图 / 空间结构更依赖 patch 的空间分布而非纯 CLS，编码器要保留足够空间信息。
