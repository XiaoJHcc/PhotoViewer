# DINOv3 照片美学评分 — Plan-3-1 数据地基（M1）

> 上游宪法：[Plan-3-0](dinov3-photo-ranking-plan-3-0-charter.md)（前提 / 意图 / 已锁定技术决策）
> 本期任务：**M1 — 数据地基 + 特征可行性**。这是整个三期的地基：先验证特征、把数据备齐、看清真实分布，再谈训练。**M1 没跑完不锁 M2 起的细节。**

## 0. 为什么 M1 必须先行、且独立成期

- 后续一切（配对、绝对校准、训练、评估）都依赖 M1 的产物与 audit 结论；M1 的真实数据分布出来前，M2+ 的阈值、配对量、绝对涌现都是猜测。
- 几个"开工前必须落实"的条件只有 M1 能确认：**星级可读性、ViT-S 特征是否够敏锐、磁盘 / 覆盖率**。任一不过，方案要改。
- 所以 M1 自成一期、带硬 GATE；跑完回看 plan-3-0 §2，再列 plan-3-2（M2 起）。

## 1. M1 步骤与 GATE

### 1.1 星级可读性确认（最先做，可能阻塞一切）

历史星级是这套训练数据的标签来源，必须先确认能读到。

- 抽样若干历史照片（含纯 RAW / RAW+JPG / HEIF 各种组合），用 `Tools/ExifTestTool`（或 BatchIngest 的 EXIF 路径）确认人工星级确实写在 **XMP / 同名 `.xmp` sidecar** 上、能被读到（ARW 的 rating 通常在 sidecar，不在文件内）。
- 顺带确认拍摄时间字段齐全（`DateTimeOriginal` / `SubSecTimeOriginal`，指纹与拍摄段切分都依赖它）。
- **GATE**：读不到（星级只在 Lightroom / Capture One 的 catalog 里）→ **先批量导出 XMP / sidecar**，否则后续全部无从谈起。

### 1.2 特征可行性：线性探针（先验证、别先炼丹）

在投入全量入库前，先确认现成 backbone 的特征对"段内细差"够敏锐。

- 取 ~200 张含多组近重复的样本，入库拿 CLS（ViT-S/16@518，384d）。
- 在"人眼可分但星级不同"的**段内可比对**上，用逻辑回归 / 线性 SVM 对 CLS **差异向量**做二分类（判谁星级更高）。
- **判据**：准确率 **≥ 80%** → 基座够用，锁 ViT-S 全量入库；**< 80%** → 先升 ViT-L（或 518→1024）重提这 200 张复测，仍不行再排查特征本身。
- 产出：`Tools/feature_probe.py` + 一页结论（含 t-SNE 目视：近重复是否聚簇、跨题材是否分离）。
- **GATE**：探针达标 → 1.3；不达标 → 切 backbone 再来（决定 plan-3-0 §1 决策 8 的最终落点）。

### 1.3 选子集 + 全量入库

- 按**题材规则**圈定训练 / 测试子集（plan-3-0 §1 决策 11；配比由你定，控泛化 / 特化）。开工前先确认：
  - 磁盘余量（两万张 patch ≈ 30GB）；
  - 库目录的**拍摄日期 / 拍摄段可恢复**（按段 / 日划分、滑窗都依赖它）。
- 跑现成 `Tools/BatchIngest`（DirectML 加速）批量入库 → 填满 `photos.db`。
- **GATE**：抽查入库完整性——指纹聚合正确（同次曝光 RAW/JPG/HEIF 合一）、EXIF / rating 覆盖率达标、CLS / patch / cv_grid 三路齐备。

### 1.4 data_audit（看清真实分布、校准所有阈值）

新建 `Tools/data_audit.py`，读 `photos.db` 输出（这是 M2+ 一切阈值的来源）：

| 统计项 | 用途 / 校准什么 |
|---|---|
| 星级分布直方图（总体 + 每段） | 确认锦标赛结构；验证"每段分布大致相同"（plan-3-0 §0.3） |
| CLS cosine 全库直方图 + 分位 | 校准相似度→权重映射、tie 上限 |
| 段内 vs 跨段 cosine 分布 | 验证滑窗 + 相似度能否分离可比对 |
| 拍摄段切分结果（按时间间隔 / 相似度） | 看每段张数、段时长，定窗口大小 |
| 近重复对（cosine ≥ 0.98）的 CV 距离分布 | 校准"条件保留"的 CV 差异阈值（决策 1） |
| EXIF / rating 字段覆盖率 | 确保 ≥ 95%，定缺失策略 |
| ≥3★ 实际占比 | 校准"Top 12.5%"是否对得上真实分布（不对则调召回口径） |

- **GATE（M1 总闸）**：星级可读 + 特征敏感 + 数据覆盖达标 + 分布无异常 → **M1 完成，进 plan-3-2（M2）**；任一异常 → 停下排查，不带病进训练。

## 3. 后续里程碑（简述，随 M1 结果调整，细节不锁）

> 只记方向与 GATE，承 plan-3-0 §1 已锁定技术决策；具体参数、配对量、架构待 M1 audit 后在 plan-3-2 起逐一展开。

- **M2 跨段绝对校准（偏移法）**：拍摄段切分 → 每段抽代表 + 精修锚点入"绝对池"跨组评级 → 拟合每段偏移 → 产出绝对星级（连续、不 clamp）。**GATE**：偏移自洽（重复评级一致）+ 跨天"同绝对级"能对齐；否则退回纯相对、绝对留 v2。
- **M3 训练对生成**：段内对（高权，相似度加权 + 条件保留 + 大星级差降权）+ 跨段对（绝对星级派生、**低权**）；候选组为单位存储；按段 / 日划 split。
- **M4 基线 + 绝对涌现测量**：段内 ranking 立地基（段内 top-1、段内 0-5 准确率）；金标准 / 锚点测**绝对涌现**（好/坏天气片落位是否合理、锚点是否落顶端）→ **a/b/c 决策点**（绝对够 → 进 M6；不够 → M5）。
- **M5 绝对校准增强（条件触发）**：绝对不够才加大偏移监督权重 / 补少量 targeted 跨段标注；重测绝对涌现。
- **M6 系统集成 + 全谱评估**：模型 + 去重压低算法 + 窗口召回；两条线评估（≥3★ 召回 + 0-5 准确率）+ 七类场景抽检 + 金标准集。**GATE**：达标 → M7，否则失败分析回退。
- **M7 部署**：ONNX 导出 + `PhotoViewer/Core/AI/` C# 推理集成（A2 已打通端侧链路，主要是换最终模型）。

## 5. 并行长周期任务：金标准验证集（建议 M1 期间就开始）

M4 起的评估、a/b/c 决策都以它为准绳（plan-3-0 §0.6 称其"唯一可信标尺"），且只能人工精选、周期长——**别等到 M4 才攒**。

- 规模 ~50-100 组，按 plan-3-0 §0.5 七类场景分布。参考构成：
  - ~30 组**近重复**（每组 3-8 张，标出最佳 1-2 张）
  - ~20 组**一般场景**（标 ≥3★ 该保留哪些）
  - ~20 张**孤立片**（标是否值得进人工二审）
  - ~10 组**极相似**（标"不可分 / tie"）
- 另含**精修精品锚点**（plan-3-0 §0.4）：既是金标准顶端，也是 M2 偏移法绝对池的成员。
- 这套集子上算的 0-5 准确率 / 组内 top-1 / 七类场景表现，比跑在带噪星级上的自动指标可信得多。

## 6. 后续里程碑的初期想法（不锁，供 plan-3-2 起草用）

> 对话中成形、但需 M1 数据 / 审计后才能定稿的想法，先记下免得丢失。**均未锁定**，plan-3-2 起按真实分布重定。

### 6.1 M2 评分偏移法（跨段绝对校准）

- **粒度**：偏移按**拍摄段**给（不按日历日）；段切分用 plan-3-0 §1 决策 2 的"时间间隔 + 相似度"那套，与可比域同源。
- **估偏移的办法**：每段抽几张代表，连同**精修精品锚点**一起丢进一个**绝对池**做一次跨组评级（小型 meta-锦标赛）；某段代表在绝对池里的级别 − 它原本的段内级别 = 该段偏移。精修片天然钉住高端。
- **刻度**：偏移后用**连续刻度、不 clamp 回 0-5**（好天 5★+2=7、坏天 0★−2=−2 是对的，表达更宽的真实绝对范围）。
- **怎么喂训练**：段内对继续用**原始段内星级、高权**（可靠地基）；跨段对用**偏移后绝对星级派生、低权**（粗信号）。
- **诚实边界**：偏移是手工注入的绝对判断、±2 很粗，质量上限卡在这——它是**锚不是真值**。M2 GATE 必须验：重复评级是否自洽、跨天"同绝对级"能否对齐；不行就退回纯相对、绝对留 v2。

### 6.2 M4 单图编码器架构初步想法

> 承 plan-3-0 §1 决策 6（纯单图）、决策 7（先线性探针）。具体维度 / 层数待 M1 探针 + audit 后定。

- **三路融合的核心概念**（来自一 / 二期）：**patch token 作为 CV 网格的 per-cell 语义权重**——每格"该不该锐"由 patch 语义定（主体格重、背景格轻），每格"实际多锐 / 多抖"由 CV 物理量给。这正是存 patch token 的理由，对应场景 6（题材化权衡）。**不**在架构里硬编码"主体清晰度权重 > 背景"，让 MLP 自己学。
- **单图编码起点**（轻量优先、防过拟合）：CLS(384) + CV 网格按 patch 主体权重做加权统计（每标量的加权 mean / std / 分位 + 最差格）+ EXIF 向量 → 小 MLP → score。先跑这个轻量版；若失败集中在"局部抖动 / 虚焦"（空间信息丢失）再上保留更多空间结构的变体。
- **EXIF 向量化**：对数化再标准化（log 焦距×crop、log 光圈×crop、log 快门、crop），缺失填 0 + mask 位；约 4-8 维。
- **判断优先级的架构含义**（plan-3-0 §0.6）：构图 → 内容 → 光影；构图 / 空间结构更依赖 patch 的空间分布而非纯 CLS，编码器要保留足够空间信息。
