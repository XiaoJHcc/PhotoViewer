## 0. 需求锚点

> 本章是项目宪法：只陈述**面临的问题**、**已有的条件**与**目标意图**，不谈技术路线。架构、损失、阈值、工具等见 §1 与下游执行计划。
> 当用户最新意见与本章冲突时，需要向用户**提出冲突**。但当用户新意见更合理时，**宪法允许补充、修正成用户意见**，但拦截不合理需求。

### 0.1 用户与拍摄习惯

- 摄影爱好者，SONY A7C2，多拍选优工作流。
- 单次外拍 ~600 张/天；同地点多角度、多焦距（24/35/85 等）、多构图反复尝试，因此大量照片高度相似。
- 历史积累上万张照片，且已全部完成人工星级标注（0-5 星）。

### 0.2 面临的问题（要替代的工作量）

- 人工筛选一次外拍约 **5 小时**，是这个项目要替代的核心成本。
- 两类最耗神、最易审美疲劳的工作（都最头疼，无所谓主次）：
  1. **在一大堆高度相似的照片中选出更好的那张**——靠相对区分。
  2. **在一大堆平平无奇的照片中挑出氛围优秀的**——靠绝对的美学直觉，反而更难。
- **整体质量水平随天气/光线真实地偏高偏低**：好天气的 1 星可能比烂天气的 3 星更好看（这是真实质量差异，不是评分噪声）。所以筛选必须在局部上下文里比较、不能用一把全局标尺，否则烂天气日会被整体埋没。
- 产品上不必精确复刻每张的星级，只需**可靠地把需要人工细看的量从 600 张压到 ~12.5%（3 星及以上）**——以经验，筛到 3 星就足以排除大部分雷同照片。

### 0.3 标注的由来与数据性质（根本前提）

**星级是怎么来的（真实选片流程，不可重新打标）：**

- **滑动窗口比对**：把同时间段/同场景的照片放一起两两比较，不跨场景跨日期。
- **每轮 Top 50% 晋级**：0→1→2→3→4→5 星共 5 轮；本星级与更高星级的照片数量大致相等（锦标赛结构）。
- 各层人工关注点不同，但已混在一起，拆不开：
  - **0→1 星**：清晰度、有效性（不是乱拍的）；多张相似中挑较好的一半
  - **1→2 星**：开始考虑美学氛围，或多张 1 星中选优
  - **2→3 星**：雷同内容选优（同拍摄点/主体/角度中最好的一张），也有孤立但氛围好、极具代表性的
  - **3→4 星**：同题材不同视角选优，且美学价值较高
  - **4→5 星**：全局精品

**这批数据的本质：**

- 它不是干净的绝对美学分，而是**摄影师在局部上下文中多轮晋级后的"生存结果"**——混合了去重选优、技术淘汰、氛围直觉、稀有瞬间保送、审美疲劳下的局部决策。
- **数据拆不开，任务也拆不开**：标注当初没有可描述的分项理由，事后也无法重标，接受它是一份混合偏好数据，不强拆成"技术分/美学分/去重分"。
- **星级是带噪声的偏序，不是严格序关系**：受逐层晋级的路径依赖影响（某张第一轮因随机原因被淘汰就永远进不了高星），跨大星级差、跨场景的比较可信度更低。
- **含极少量随机噪声**：像素级几乎一致、人工只能随机择一晋级的极相似对，其星级差不携带审美信息；仅当两张存在真实可辨的差异（锐度、抖动、曝光等）时，这种近重复才是有价值的强监督。
- **孤立照片也是"比"出来的**：所谓无相似可比的孤立照片，其星级同样来自当天临近照片的比较，符合当天的均值范围；只是这种可比性人工讲不清（低相似、跨内容）。所以同天的低相似照片之间存在**真实但难解释的弱偏好**——它们不是不可比，而是弱可比，这也是孤立照片能进入训练的依据。
- **0★ 多是"美学价值极低的正常照片"，不是技术废片**：明显的技术废片已删掉一部分（少量抖动片可能仍在）。所以模型在低分端学的是**美学下限**，而非废片检测；技术废片交给 CV，产品也不靠模型去捡已删的垃圾。**由此产生训练/推理分布错位**：训练库是"删除后的幸存者"，而产品推理面对的是外拍原始 dump、含模型从未见过的废片类型（严重虚焦、误触快门、极端曝光等）——模型对它们打分是未定义行为，必须由模型外的**废片闸门**先行拦截（见 §1 决策 14）。
- **星级是按场景归一化的相对排名，不是绝对质量**：每个拍摄段独立做锦标赛，所以**每段的星级分布大致相同**（好天气顶多让晋级比例从 ~50% 微浮到 60%），与天气基本无关。天气真正改变的是照片的**实际质量水平**（好天气整体更好，真实差异、非噪声）——于是**同样是 3★，好天气的明显优于坏天气的，但标签一样**。一句话：**标签里有强的"段内相对"信号，却几乎没有"跨段绝对"信号。** 个人标准跨年稳定（随机性远小于天气），所以绝对靶本身是稳的，只是标签没标出来——见 §0.6 挑战 3。
- **星级里掺了"后期潜力"的考量，不是纯直出审美**：用户对**构图 / 内容**的权重高于**影调**，正因为**影调可后期修复、而构图 / 内容不可救**；天生好影调仍加分（锦上添花）。所以同一份标签里"as-shot 审美"与"潜力感知偏好"是**混合**的——典型表现：雾天发灰的片子拿高星很常见（看中其构图与潜力，而非当下影调）。
- **雾 / 低对比片的星级建立在"增强后视图"上**：用户选这类片时会**先用 ACR 拉对比 / 去雾才看得清细节**再判分（人眼在原始低对比下也看不清）。所以这些片的标签对应的是增强视图、不是平直直出——这是 plan-3-1 给模型补"确定性增强视图"的直接依据。

### 0.4 已有的条件与可用信号

- **算力**：本期训练只需在 4080 桌面端跑得动，内存/算力放开用；端侧（iPad/移动端）下放到远期。
- **每张照片可用的信息（已提取并入库）**：
  - 整图的**视觉语义特征**（全局语义向量 + 每格 patch 语义先验，含构图、内容、色彩氛围的高维表达）；
  - 原始分辨率下全画面网格采样的**物理质量图**（锐度、抖动方向/程度、曝光对比度等，专门补足视觉模型在缩略图尺度看不见的像素级技术差异）；
  - **EXIF 拍摄参数**（焦距、光圈、快门、CMOS 倍率等；焦距本身就改变评价标准——24mm 与 35mm 的取舍不同）；
  - **人工星级**（0-5）。
- **同次曝光的 RAW/HEIF/JPG 三联共享同一身份指纹**，特征只算一次、可跨格式复用。
- **相似性本身是一种条件**：照片间的视觉相似度既可用于训练时加权（越相似的对越有可比性、越该重点学），也可用于推理时聚类去重——它是连续的程度，不是非此即彼的归类。
- **已有工程基建**：照片特征数据库、指纹计算、全文件夹批量索引器、离线批量入库 CLI 均已落地，可直接产出训练数据。
- **星级随原始照片文件走**：rating 在文件 / sidecar 上可直接读；**训练/测试子集尚未圈定**，按题材规则选取（下条），不是全量无脑入库。
- **题材配比可控**：按需要泛化或特化，给各题材指定数据量、按规则选数据集——这圈定了模型的能力覆盖。
- **精修这一"动作" = 顶层标签信号（不是把精修像素喂模型）**：精修过的照片其**直出原片已在库（本就是高星）**；"我愿意花时间精修它"是极强的"顶级候选"人工偏好。**用法**：把"是否精修"作为打在**直出原片**上的高权顶层标记，而非把精修后的像素当输入——精修像素是 OOD、推理期复现不了（同 §1 决策 13 增强铁约束）。**注意覆盖不均**（好天气近乎全精修、坏天气仅半数 5★ 被精修）→ 它带天气偏置，只能当"段内顶层"弱信号，**不能拍平成统一"六星"**；跨段绝对仍由评分偏移（§1 决策 3）负责。精修版可作搭建金标准 / 回忆顶级片时的**参照**，不作模型输入。**前提待确认**：精修成品是 ACR 导出件、散落在库外，文件名 / 尺寸 / EXIF 可能已变、指纹大概率对不上——能否可靠回溯匹配到原片指纹是 M1 前置确认项（plan-3-1 §1.1）；匹配不可行则本条信号降级、绝对校准的腿重议。
- **模型输入与打分对象是直出渲染（推理期只见直出）；但标签掺入了后期潜力考量**（见 §0.3）：构图 / 内容须本身就好（不可后期救）、影调因可救而降权但天生好仍加分。由此产生"标签奖励了模型看不见的潜力"的**有界噪声**，靠软权重吸收、且这类片多由构图扛着；雾 / 低对比片更需给模型确定性增强视图（plan-3-1）。另：对焦点坐标当前不在可用条件里（Sony MakerNote 解析二期已推迟）。

### 0.5 模型必须应对的典型场景（能力要求 / 验收基线）

| # | 场景 | 模型必须做到 |
|---|---|---|
| 1 | **同山头不同焦距/构图** — 24mm vs 35mm，偏左 vs 偏右 | 判断"留白呼吸"加分 vs "囊括杂乱树枝"减分 |
| 2 | **同题材长焦梯田** — 风格内容几乎一致，只是田埂曲线 / 小房子位置不同 | 判断抽象视觉元素在构图中的构成美学，而非题材喜好 |
| 3 | **沿山脊移动** — 前景变了远山没变 | 判断同题材下差异部分的美学优劣 |
| 4 | **静物连拍** — 两张几乎一样，仅像素级差异 | 凭技术质量（对焦、抖动等视觉模型看不见、已由物理质量图补足的信息）分辨；真无差异则判不可比，不强行二选一 |
| 5 | **孤立瞬间抓拍** — 氛围/光线极佳但无相似可比 | 不依赖对比直接判断美学：有 5 星潜质的至少给到 3 星（满足 3 星召回），但不过度预测 |
| 6 | **主体 vs 背景的题材化权衡** — 人像要主体锐+背景虚，风光要全清+接受留白，有些题材可实可虚 | 根据题材语义切换权衡，不是统一规则 |
| 7 | **夜景抖动判断** — CMOS 防抖失败，部分清晰部分拖影 | 结合物理质量图与语义，重点识别建筑边缘 / 点光源、避开车流/树叶等天然变化区，判断照片各处是否该实、该虚 |

任何阶段的中间产物或最终模型，都必须能在每一类的代表样本上给出合理输出；验证集设计直接对齐这 7 类。

### 0.6 核心目标与成功判据

**唯一目标：让模型真正理解摄影美学，对任意一张照片给出反映其美学水准的分数——包括没有相似可比的孤立照片。**

- 这正是 3 星以上评分的难点所在：按人工习惯，**3 星以上已无太多雷同**，靠的不是去重，而是**绝对的审美判断**。绝对美学理解才是这个目标的最高层次。
- 它是**单一能力、单一分数**：同一个分数既要在相似组里分得开，也要对孤立照片站得住——不拆两个头、不分主次任务。
- **审美的判断优先级（人工次序）**：**构图**为首 → **内容**（人物姿势、车船位置等可遇不可求的瞬间）→ **光影 / 氛围 / 色彩**。前两者是大头、覆盖各星级；光影氛围多是 3-5★ 的精细评价——产品上未必关键，但模型要能认到"好氛围"才能召回 3★ 氛围片。影调排末位是因为**它可后期修复**（构图 / 内容不可救）；但"天生好影调"仍是正信号、要认得出（召回 3★ 氛围片）。"构图优先"与"认得出影调"**不是二选一**——靠混合标签让模型学出软权重即可，不靠破坏性抹除影调强行实现（见 §0.7、§1 决策 13）。

**实现这个目标必须跨过的核心挑战（不是并列任务，而是同一目标的不同难点）：**

1. **相似组内的区分度 + 去重的分工**：高质量连拍若纯按美学本就该整组高分（合理），但这既不符合产品（不重复展示）也不符合数据分布（锦标赛减半）。分工是：**模型**只在**真实可辨的差异**（对焦、抖动、轻微构图）上压低，**不**为凑"减半"去硬排等价连拍（否则污染绝对美学）；"等价高质连拍只留一张"交给**去重压低算法**，它在**训练侧**（等价对判 tie 不监督）与**推理侧**（产品只亮 top-1）双侧与模型配合。相似组也恰是数据里监督最强的信号、训练的主要抓手——但**是手段与挑战，不是最终目标**。
2. **低相似/孤立照片的绝对评价**：两张不相似时很难讲"谁更该晋级"——但它们其实**在当天被比较过**（孤立照片符合当天均值范围，见 §0.3），只是这种可比性人工讲不清。所以这里是**弱可比、难解释**，不是无可比；模型既要利用这层弱监督，也要能靠对画面本身的理解给出绝对分。这是更难、更接近"真正理解审美"的部分，是 3 星以上召回的核心。
3. **绝对标尺不在标签里——必须从别处引入**：星级按场景归一化（见 §0.3），给的是**强的段内相对信号、几乎为零的跨段绝对信号**；而目标要的恰是跨段绝对分（孤立片、跨天比较、氛围召回都靠它）。**确定的陷阱**（不是风险）：按通常规则训练会把"好天气 3★"和"坏天气 3★"当等价（标签一样、实际优劣明显），**直接污染绝对尺度**。所以——① **绝不用星级跨段组对**（段内相对可信，跨段星级不可比）；② 跨段绝对校准只能从**非标签来源**引入：主要是**精修精品锚点**（"跨段都算顶尖"的绝对信号，正好补标签所缺）+ 视觉特征的泛化，并用金标准集验证。**诚实定位**：可靠的能力是**段内相对排序**；**跨段绝对分是"特征 + 锚点"的尽力外推**，是这个目标的核心不确定性——产品因此也要靠窗口（在可靠的段内信号上做召回）兜底，而不是把分数拍平 / 做天气归一化。

> 一句话：**相对比较是训练的手段、是必须过的关；绝对美学分是要交付的结果。** 二者不矛盾——用相对监督，学出一个全局一致的绝对分。
>
> **锦标赛逐层 Top 50% 是采集标签的方式，不是模型要复刻的流程**：模型从星级反推审美偏好即可，是否再按工作流分桶是输出环节的事。

**保底交付形态（结构性安全垫，封顶下行风险）：**

人工标签本来就产自"段内锦标赛、局部 Top 50%³"——所以"**段内 ranking + 每段 Top 12.5% 配额 + 去重压低 + 废片闸门**"这套纯局部系统，**在构造上就复刻了人工选片过程本身**，不依赖任何跨段绝对分即可逼近产品门槛（600 → ~75 张人工细看）。跨段绝对分负责的是**上行空间**（跨天比较、好天气配额加码、氛围片超配额召回、单图模式）——是增量，不是生存条件。挑战 3 实测不达标时，项目**降级为保底形态交付**而非失败；残余风险集中在场景 5（孤立氛围片）的段内排序质量本身。

**成功判据：**

- ≥3 星召回率 ≥ 80%（人工只需看 Top 12.5%）——**产品阶段门槛**，主要考验孤立/低相似照片的**绝对**美学判断。
- **0-5 星评定准确率**——**判模型能力、定优化方向的全谱标尺**：把"模型分 + 雷同压低算法（及未来其他算法）"组成的完整系统还原成 0-5 星，与人工 0-5 标签比逐级准确率 / 序相关。≥3★ 召回只是门槛；要持续衡量"模型 + 配套算法"的综合能力强弱、找优化方向，最终回到这条。注意在局部可比域 / 金标准集上算更可信（全局星级带漂移噪声）。
- 相似组内最优得分最高、且 ≥3 星里不含太多雷同——考验**相似组区分度**（挑战 1）。
- 不被整体美学均值浮动带偏：好天气日的平庸片不该挤掉烂天气日的组内最佳（按拍摄时段分桶校验）。
- 判别能力以一套人工精选的金标准样本为准绳——能逐类应对 §0.5 七类场景，才算真的有用；自动指标跑在带噪星级上，容易"loss 好看但没用"。
- **商业意义上的成功** = 5 小时人工 → 半小时人机协作；不被"全自动精确打分"的执念带偏。

**产品形态**：人机协作——高置信度自动保留/淘汰，低置信度回人工复核；模型需要能表达"自己有多确定"。

### 0.7 必须守住的原则（约束）

- **比较只在局部可比域内发生，且可比性按"场景实际变化"算、不按钟表时间**：同样隔 30 分钟，傍晚光线变化远大于正午——所以**内容相似度**是可比性主信号（它直接度量变化，傍晚相似度掉得也快），时间只用来给序列**排序、切分拍摄段**，不当线性距离。可比域取**时间序列上按张数滑动的窗口**（模拟人工滑窗、自动适配拍摄密度），窗口内低相似对软降权不硬排除（孤立照片靠此进训练）；绝不跨拍摄段 / 跨天硬组对——那种比较从未真实发生，只会注入场景偏置。
- **模型必须学到可迁移的审美，而非记忆特定场景**。数据按拍摄日期/事件划分，同次拍摄绝不跨 train/val/test。
- **标签是带噪声的软偏好，不是绝对真值**。监督带可信度权重，不是"星高必胜"的硬约束；可信度低的比较降权或剔除。
- **极相似的随机摇号是噪声**：只有当存在真实可辨差异时，近重复对才作为有效监督。
- **软权重、数据驱动，不堆人工硬规则**：不把"主体该锐、背景该虚"这类经验写成 if-else，让模型从数据里学权衡。
- **局部信息来自全画面客观采样**，不寄望显著性热力图恰好落在审美关注点上。
- **不做对星级的绝对回归/分类**（跨场景同星级不可信，硬拟合必学成平均分）——要交付的"绝对美学分"是用相对比较缝合出来的全局尺度，不是直接拟合 0-5 数值。
- **不追求完全替代人工，不一次性堆统一大模型**——先把最核心的审美判别做扎实，复杂结构留到基线验证后按需增量。
- **允许给模型"确定性、可复现"的增强视图，但不破坏性抹除影调**：对雾 / 低对比这类"人眼也需增强才能判"的照片（§0.3），可用确定性算法生成增强视图、与原片并行喂入（多视图），让"构图可辨"与"影调理解"兼得；但绝不用全局直方图均衡等手段破坏性抹掉影调，绝不用**不可复现的手动修图**作输入。优先级靠软权重**学**出来，不靠抹除**做**出来。

## 1. 已锁定的技术决策（贯穿各期，M1 实测后只微调阈值，不改方向）

> 这些是经多轮讨论收敛的技术路线锚点，独立于 §0 的纯意图；任一执行计划（plan-3-1 起）都须落实。带"待 audit"的只调参数，不改方向。

1. **极相似对条件保留**：CV/EXIF 有可测差异（锐度/抖动/曝光）才留并高权；embedding 与 CV 都近乎一致 → 判真 tie 丢弃。
2. **可比域 = 按张数滑动窗口 + 内容相似度为主**：窗口在时间序列上按张数滑（模拟人工滑窗、自动适配傍晚快变）；可比性权重以内容相似度为主，时间只用于排序与切分拍摄段、**不做线性 time_decay**；同段低相似对软降权不硬排除（孤立照片靠此进训练）。**禁止用原始星级跨段组对**（标签按场景归一化、跨段不可比）。
3. **跨段绝对校准从非标签来源引入**：标签只给段内相对，跨段绝对靠——（a）每段手动**评分偏移**把段内星级抬成近似绝对星级（连续刻度、不 clamp，跨段对用它派生、低权）；（b）**精修"动作"作直出原片的顶层标签信号**（非精修像素入模、覆盖不均故非统一"六星"，见 §0.4）；（c）视觉特征泛化；（d）**备选**：训练时把每段偏移作为**可学习自由参数**（score = f(x) + b_seg，推理期丢弃 b_seg；Bradley-Terry 评委偏置的标准做法）——段均值被 b_seg 吸收、f(x) 只能用视觉特征解释跨段差异，人工偏移退居"锚定尺度 + 验证自洽"，(a) 的人工量可大幅压缩；(a)/(d) 配比待 M1 绝对性探针（plan-3-1 §1.5）后在 plan-3-2 定。以上均用金标准集验证。
4. **监督是带可信度的软偏好**：星级差、相似度、时间临近共同决定权重；大星级差（跨 3 星以上、多为跨场景）**降权**而非加重；不做"星高必胜"硬约束、不做对星级的绝对回归。
5. **去重压低是模型外的协同算法**：训练侧等价对判 tie 不监督、推理侧产品只亮 top-1；模型只在真实可辨差异上压低，不为凑"减半"硬排等价连拍。
6. **第一版纯单图 → score，不做邻域聚合**；邻域聚合 `[h_i; g_i]` 下放条件触发，Transformer 更远期。
7. **架构定型前先线性探针**验证特征可分性（≥80% 即基座够用）。
8. **v1 训练 backbone = ViT-S/16@518**（`dinov3_vits16_f32_518_v1`，与入库链路一致）；评估场景 2 细节判别不足再升 ViT-L 重提（`(fingerprint, model_id)` 共存，不返工 schema）。**特征可分是"改到成立为止"的环节，升级梯四级封顶**：ViT-S/16@518 → ViT-L → 更高分辨率（518→1024）→ **LoRA 微调 backbone**（最终逃生梯，4080 跑 ViT-S LoRA 无压力）——有第四级兜底，"特征不够敏锐"只是成本问题，不是项目风险。
9. **置信度走事后估计**（窗口间打分方差 / 到决策边界距离 / 邻居分歧），不训伪造目标的置信度头。
10. **评估两条线**：≥3★ 召回 @ Top 12.5%（产品门槛）+ 0-5 星评定准确率（段内比对 + 金标准/锚点，因星级跨段不可比；系统级、含去重算法）作为能力标尺；并按拍摄时段分桶查召回（抗均值漂移）、对齐 §0.5 七类场景。
11. **训练/测试子集按题材规则选取**（配比控泛化/特化）；按拍摄日期/事件划分 train/val/test，同次拍摄绝不跨 split；**金标准集样本所在拍摄段必须划入 test（或至少排除出 train）**，否则绝对涌现测量被污染。
12. **数据以"候选组/拍摄段"为单位组织**（可派生 pairwise/listwise，换损失不重做数据工程）。
13. **模型输入增强：确定性 + 推理期可复现、多视图兼得、禁破坏性抹除**：为补"雾 / 低对比片人眼也需增强才能判"（§0.3），允许给 DINO 喂"原片 + 确定性增强视图"的**多视图**输入（构图可辨 + 影调理解兼得）；增强算法须确定性、推理期对每张可复现，**手动修图只作调参参照、不作输入**；**禁全局直方图均衡等破坏性抹影调**；不用"不变性增广"（会杀影调）、不用调色版当更高星新样本（OOD）。**具体算法 / 参数 / 产品化待 plan-3-1 §1.2 探针校准**（属"待 audit"类，只调参数不改方向）。
14. **废片闸门是模型外的第二个协同算法**（与决策 5 去重压低并列）：训练库不含已删废片（§0.3 分布错位），模型不负责废片检测；产品推理链路里由 CV / 规则闸门（抖动、严重虚焦、极端曝光等）**先行拦截再进模型评分**。验收样本须含**未删的原始废片**；后续外拍保留原始 dump 作闸门校准与金标准废片组的数据来源。

> 阈值类（cosine 下限、tie 上限、窗口大小、相似度→权重映射、偏移幅度等）一律由 `data_audit` + 人工核验校准，不预设。
