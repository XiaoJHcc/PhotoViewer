## 1. 绑定的宪法决策（plan-3-0 §1，本册必须落实）

决策 **5 推理侧**（去重压低：产品只亮 top-1）、**9**（置信度分流的产品形态：高置信自动保留 / 淘汰，低置信回人工复核）、**10**（评估两条线：≥3★ 召回 @ Top 12.5% + 系统级 0-5 星评定准确率）、**14**（废片闸门先拦再评分，验收含未删原始废片）；部署契约承决策 **8**（与入库同链路的 backbone、ONNX、`(fingerprint, model_id)` 版本化失效）。

## 2. 本册必须回答的问题（决策清单）

1. 去重压低推理侧参数：等价判定的 cosine / CV 阈值（承 M1 audit）；
2. **废片闸门规格**：哪些信号（ShakeFlag / CV 锐度 / 曝光极值）拦哪类废片、阈值、漏拦兜底策略；
3. 窗口召回参数：窗口张数、每窗配额——**保底交付形态的核心旋钮**（plan-3-0 §0.6）；
4. 置信度分流阈值：自动保留 / 自动淘汰 / 人工复核的三段切分；
5. 系统级 0-5 星还原规则：模型分 + 协同算法 → 星级（供全谱准确率评估与产品展示）；
6. 分数持久化：`photos.db` 是否入库模型分、失效策略是否随 `model_id`；
7. M7 部署：ONNX 导出精度（fp32 / fp16）、C# 集成落点（`PhotoViewer/Core/AI/`，A2 已打通端侧链路）、端侧范围（远期，本期桌面即可）。

## 3. 输入 / 产物 / GATE

- **输入**：最终模型 + 置信度方案（plan-3-3）、金标准集（**含未删废片组**）、M1 audit 阈值表。
- **产物**：完整系统（模型 + 去重压低 + 废片闸门 + 窗口召回 + 置信度分流）；全谱评估报告；ONNX 模型 + C# 推理集成；一次真实外拍的端到端试跑记录。
- **GATE（M6）**：≥3★ 召回 ≥80% @ Top 12.5% + 系统级 0-5 准确率达标 + 七类场景逐类抽检 + 按拍摄时段分桶无挤占（好天平庸片不挤掉坏天组内最佳）+ 废片闸门在未删废片组上验收通过；不达标 → 失败分析**定位到层**（模型 / 哪个协同算法），回对应册修复。
- **GATE（M7）**：桌面端真机端到端跑通——新外拍 dump → 批量入库 → 评分 → 召回展示，无崩溃（对齐 CLAUDE.md §9 实机验证要求）。
